Workflow

How the workflow is intended to run

Procedure Suite scrubs PHI in the browser, then runs extraction on scrubbed text only. You can process a single note and manage longitudinal events from the Local Vault (Zero-Knowledge Clinical Registry). This page documents the flow and guardrails that keep absolute dates and identifiers out of server requests.

Single-note workflow (default)

  1. Paste the procedural note into the editor.
  2. Run PHI detection and review each proposed redaction.
  3. Apply redactions (or manually add/remove them as needed).
  4. Submit the scrubbed note for CPT and registry extraction.
  5. Review codes, evidence, and extracted registry fields.
  6. Edit table values in the flattened tables panel (optional).
  7. Export JSON or tables for downstream review/training.

What’s new in the workflow

  • New Note: clears the editor, tables, and JSON output so you can start fresh.
  • Flattened Tables (Editable): expand the section at the bottom to edit values, add rows, and correct entries.
  • Edited JSON (Training): when you edit tables, a second JSON payload is generated and labeled for training.
  • Export JSON: download the raw response payload.
  • Export Tables: download the flattened tables as an Excel-readable file.
  • Add Event (Vault): append pathology, imaging, procedure, or clinical-status events with signed day offsets only.

Event chronology in Local Vault

The Local Vault stores encrypted local metadata per case (`patient_label`, `index_date`, optional local fields). When you add an event, the browser computes relative_day_offset from the local index date and event date. The server receives only the signed integer offset and event payload.

The index date never leaves your browser in plaintext. Absolute dates are not sent to append APIs.

Add Event step-by-step

  1. Unlock Local Vault and select a case row.
  2. Click Add Event.
  3. Choose Event Date and Event Type.
  4. Choose input mode: note extraction or structured status.
  5. Submit the event. The request includes only relative_day_offset (never absolute dates).
  6. For structured-only events, append persists directly without extraction pipeline calls.

Event guardrails and troubleshooting

  • If index date is missing locally, offset defaults to 0 and the UI warns.
  • Event Date is client-side only and must be valid before submit.
  • Structured-only mode requires at least one status toggle or comment.
  • Remote registry edits and local vault edits are saved via separate paths.

What happens to PHI?

PHI detection runs entirely in your browser. The server only receives the scrubbed text that you explicitly submit. For append events, the server receives only signed relative_day_offset integers, not absolute dates.
